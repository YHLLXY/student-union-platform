## 一、项目概述

### 1.1 项目定位

为学生会内部成员提供一个集**任务管理、信息公告、部门交流、活动票务**于一体的线上办公平台。风格参考钉钉/飞书，精简办公风，功能简约但能力强大。

### 1.2 核心需求

- 登录认证：姓名 + 学号 + 部门邀请码 + 密码（首次登录设置密码并确认）
- 四种角色：志愿者（普通成员）、部门负责人、主席团、校领导（老师）
- 任务发布与审核（实时动态更新）
- 部门内部消息公告
- 校级信息发布
- 个人任务完成情况统计与日历视图
- 部门内部论坛（公告板形式，先上线；后续扩展实时聊天）
- 艺术活动抢票（先到先得，抢票成功自动反馈学号与姓名）
- 权限分级管理

### 1.3 非功能性需求

| 要求 | 实现策略 |
|------|---------|
| 模块隔离 | 每个模块独立目录 + CSS Modules + 独立错误边界，一个模块崩溃不影响其他 |
| 兼容扩展 | 新增模块只需加路由和菜单配置，不动已有代码 |
| 实时更新 | Supabase Realtime 订阅，数据库变更自动推送前端刷新 |
| 运行稳定 | 三层错误处理 + 请求自动重试 + 数据库行级安全 |
| 精简办公风 | Ant Design 组件 + 深蓝主色调 + 200ms 淡入动效，无华丽动画 |

### 4.3 模块隔离原则

1. **每个模块只暴露 `index.tsx`**，其他模块只能引用这个入口文件
2. **模块内部组件和数据服务不对外暴露**，外部不可直接 import
3. **样式使用 CSS Modules**（`*.module.css`），类名编译后自动加 hash，不会冲突
4. **每个模块有独立的错误边界**，模块崩溃时显示降级 UI，其余模块正常
5. **新增模块不影响已有模块**，只需在路由配置和菜单配置中添加条目

## 五、数据库设计

### 5.1 表结构

Supabase 使用 PostgreSQL，以下为 9 张核心表：

#### `users` — 用户表
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  auth_id UUID REFERENCES auth.users(id),  -- Supabase Auth 自动管理表
  name TEXT NOT NULL,
  student_id TEXT UNIQUE NOT NULL,
  department TEXT NOT NULL,  -- 枚举值见 constants
  role TEXT NOT NULL DEFAULT 'volunteer',  -- volunteer | dept_head | presidium | teacher
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### `invite_codes` — 邀请码表
```sql
CREATE TABLE invite_codes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  code TEXT UNIQUE NOT NULL,
  department TEXT NOT NULL,
  created_by UUID REFERENCES users(id),
  is_used BOOLEAN DEFAULT false,
  used_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### `tasks` — 任务表
```sql
CREATE TABLE tasks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  content TEXT,
  priority TEXT NOT NULL DEFAULT 'normal',  -- urgent | important | normal
  status TEXT NOT NULL DEFAULT 'pending',   -- pending | in_progress | review | completed
  deadline TIMESTAMPTZ,
  created_by UUID REFERENCES users(id),
  assigned_to UUID REFERENCES users(id),   -- NULL 表示指派给整个部门
  assigned_department TEXT,                 -- 指派给部门时使用
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

#### `task_submissions` — 任务提交记录
```sql
CREATE TABLE task_submissions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id UUID REFERENCES tasks(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id),
  note TEXT,                               -- 提交说明
  status TEXT DEFAULT 'submitted',         -- submitted | approved | rejected
  review_note TEXT,                        -- 审核意见
  submitted_at TIMESTAMPTZ DEFAULT now(),
  reviewed_at TIMESTAMPTZ
);
```

#### `notices` — 部门公告表
```sql
CREATE TABLE notices (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  content TEXT,
  type TEXT DEFAULT 'notice',              -- notice | meeting | activity
  department TEXT NOT NULL,
  is_pinned BOOLEAN DEFAULT false,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### `school_notices` — 校级通知表
```sql
CREATE TABLE school_notices (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  content TEXT,
  is_pinned BOOLEAN DEFAULT true,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### `forum_posts` — 论坛帖子表
```sql
CREATE TABLE forum_posts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  content TEXT,                            -- Markdown 格式
  category TEXT DEFAULT 'work',            -- work | activity | resource | chat
  department TEXT NOT NULL,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

#### `forum_replies` — 论坛回复表
```sql
CREATE TABLE forum_replies (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  post_id UUID REFERENCES forum_posts(id) ON DELETE CASCADE,
  content TEXT NOT NULL,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### `tickets` — 票务表
```sql
CREATE TABLE tickets (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  description TEXT,
  cover_url TEXT,
  total_count INTEGER NOT NULL,
  per_user_limit INTEGER DEFAULT 1,
  open_time TIMESTAMPTZ NOT NULL,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### `ticket_records` — 抢票记录表
```sql
CREATE TABLE ticket_records (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ticket_id UUID REFERENCES tickets(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id),
  student_id TEXT NOT NULL,
  name TEXT NOT NULL,
  grabbed_at TIMESTAMPTZ DEFAULT now()
);
```

### 5.2 实时订阅策略

基于 Supabase Realtime（WebSocket），前端订阅以下表的变更：

| 表 | 订阅事件 | 触发场景 |
|------|---------|---------|
| tasks | INSERT / UPDATE | 新任务发布、任务状态变更 |
| task_submissions | INSERT / UPDATE | 成员提交任务、负责人审核 |
| notices | INSERT | 新公告发布 |
| school_notices | INSERT | 新校级通知 |
| forum_posts / forum_replies | INSERT | 新帖/新回复 |
| tickets / ticket_records | INSERT | 新票务发布、抢票成功 |

前端收到变更事件后自动更新对应模块的 UI，无需用户手动刷新。

### 5.3 数据安全策略

使用 Supabase Row Level Security (RLS) 实施行级安全：

```sql
-- 用户只能读取本部门的数据
CREATE POLICY "users_read_own_dept" ON tasks
  FOR SELECT USING (assigned_department = (SELECT department FROM users WHERE id = auth.uid()));

-- 只有负责人/主席团/老师可以创建任务
CREATE POLICY "leaders_create_tasks" ON tasks
  FOR INSERT WITH CHECK (
    (SELECT role FROM users WHERE id = auth.uid()) IN ('dept_head', 'presidium', 'teacher')
  );

-- 只有票务发布者和校领导能看完整抢票名单
CREATE POLICY "ticket_owner_view_records" ON ticket_records
  FOR SELECT USING (
    ticket_id IN (SELECT id FROM tickets WHERE created_by = auth.uid())
    OR (SELECT role FROM users WHERE id = auth.uid()) = 'teacher'
  );
```

## 八、稳定性设计

### 8.1 错误处理三层架构

```
层级 1: 全局错误边界 (App 级别)
  └── 捕获未预料的 React 渲染错误
  └── 显示友好错误页 + 刷新按钮
  └── 不会白屏

层级 2: 模块错误边界 (每个模块包裹)
  └── 模块渲染失败时，仅该模块区域显示错误提示
  └── 其余模块不受影响
  └── 提供「重新加载此模块」按钮

层级 3: 接口请求错误处理 (每个 service)
  └── 网络超时 → Toast 提示 + 自动重试 2 次
  └── 权限不足 → Toast 提示「无权限」+ 不发起重试
  └── 未知错误 → Toast 提示 + 记录到控制台
```

### 8.2 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 列表为空 | 显示 Ant Design Empty 组件 + 引导文字 |
| 网络断开 | 检测 navigator.onLine，显示离线提示条 |
| 登录过期 | 拦截 401，自动跳转登录页 |
| 并发抢票 | 数据库事务 + 行锁，确保不超发 |
| 邀请码已使用 | 前端禁用 + 后端校验双重保证 |

### 8.3 数据库事务（关键操作）

```sql
-- 抢票操作使用数据库事务确保并发安全
BEGIN;
  -- 检查剩余票数
  SELECT COUNT(*) FROM ticket_records WHERE ticket_id = $1;
  -- 检查用户是否已抢
  SELECT COUNT(*) FROM ticket_records WHERE ticket_id = $1 AND user_id = $2;
  -- 检查每人限抢
  -- 插入记录
  INSERT INTO ticket_records (ticket_id, user_id, student_id, name)
  VALUES ($1, $2, $3, $4);
COMMIT;
```

## 九、兼容扩展设计

### 9.1 新增模块步骤

以后要添加新功能模块（比如实时聊天、投票系统、活动报名等），只需：

1. 在 `src/modules/` 下新建文件夹
2. 编写模块组件 + service + CSS Module
3. 在 `src/utils/constants.ts` 中添加菜单配置项
4. 在 `src/App.tsx` 中添加路由配置

**已有 8 个模块的代码不需要任何修改。**

### 9.2 模块更新迭代

更新某个模块时：
- 修改模块内部文件（组件/样式/服务），对外接口 `index.tsx` 保持不变
- 其他模块通过 `index.tsx` 引用，不受内部改动影响
- 如需修改对外接口，只影响引用该接口的少数位置，编译时即可发现
