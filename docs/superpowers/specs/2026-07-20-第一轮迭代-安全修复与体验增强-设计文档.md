## 三、功能设计

### 3.1 密钥清理

#### DEV2026PRESIDENT → 环境变量

**改动文件：** `.env`、`.env.example`、`LoginPage.tsx`

- `.env` 和 `.env.example` 新增 `VITE_DEV_KEY=DEV2026PRESIDENT`
- `LoginPage.tsx:267` 改为 `import.meta.env.VITE_DEV_KEY || 'DEV2026PRESIDENT'`（保留 fallback）

#### 默认密码 123456 → 随机生成

**改动文件：** `adminService.ts`

- 每次重置时 `Math.random().toString(36).slice(-8)` 生成 8 位随机密码
- 返回值从 `boolean` 改为 `string | false`，返回明文密码供管理员告知成员
- 同步检查调用方是否依赖布尔返回值

### 3.2 任务搜索 + 筛选

**改动文件：** `TaskListPage.tsx`（仅此一个文件）

**UI 布局：**

```
PC 端（一行）：
┌──────────────────────────────────────────────────────────┐
│ 🔍 搜索任务标题...    │ 优先级 ▼ │ 部门 ▼  │  [重置]    │
└──────────────────────────────────────────────────────────┘

移动端（两行，<768px）：
┌──────────────────────────────┐
│ 🔍 搜索任务标题...           │
│ [优先级 ▼    ] [部门 ▼     ] │
│ [重置]                       │
└──────────────────────────────┘
```

**实现方式：**

- 客户端过滤，不改 Service 层
- 搜索框 `Input.Search`，300ms 防抖（使用 `useEffect` + `setTimeout`）
- 优先级：antd `Select`，选项 全部/紧急/重要/普通
- 部门：antd `Select`，选项 全部 + DEPARTMENTS 常量 12 个部门，仅 `hasMinRole(role, 'president')` 可见
- 重置按钮清空所有筛选条件（搜索词 + 优先级 + 部门 + Tab 状态）
- 扩展现有 `useMemo` 的 `filteredTasks` 逻辑

### 3.3 月报逾期修复

**改动文件：** `dashboardService.ts`

**根因：** 第 353 行和第 378 行直接 `overdue: 0`，无真实查询。

**修复：**

```typescript
// 第 353 行（部门维度）：在 for 循环内追加
if (t.status !== 'completed' && t.deadline && dayjs(t.deadline).isBefore(now)) {
  deptStats[d].overdue++;
}

// 第 378 行（总览维度）：改为
const totalOverdue = allTasksRes.filter(
  t => t.status !== 'completed' && t.deadline && dayjs(t.deadline).isBefore(now)
).length;
```

- 零新增查询，复用已有的 `allTasksRes` 数据
- `dayjs` 已在文件顶部导入
- `now` 变量已在函数顶部定义（`const now = dayjs()`）

### 3.4 任务可编辑

**改动文件：** `taskService.ts`（新增函数）、`TaskDetail.tsx`（新增 UI）

#### Service 层

```typescript
export async function updateTask(taskId: string, updates: {
  title?: string;
  content?: string;
  priority?: string;
  deadline?: string | null;
  assigned_to?: string | null;
  attachments?: Attachment[] | null;
}): Promise<boolean>
```

- 不可编辑字段：`assigned_department`（部门归属不变）
- 更新 `updated_at` 时间戳

#### UI 层

- 编辑按钮：TaskDetail Modal 标题栏右侧，`Button` + `EditOutlined` 图标
- 可见条件：`created_by === user.id || isAdmin(user.role)`，且 `task.status !== 'completed'`
- 编辑 Modal：antd `Form`，预填当前值，字段与 TaskForm 创建表单一致（减去部门选择）
- 保存 → `updateTask()` → `message.success` → 关闭 Modal → 刷新 `onUpdate()`
- 移动端：Modal `width={md ? 600 : undefined}`
