> 版本：v1.0  
> 日期：2026-06-29

## 一、背景与目标

### 1.1 痛点分析

当前平台已覆盖学生会日常办公的 8 大模块，但在实际运行中主席团反馈以下核心问题：

| 痛点 | 表现 | 根因 |
|------|------|------|
| 换届知识断层 | 老成员离任后，新人不知过往工作流程、历史决策、对接人信息 | 任务无模板沉淀、工作记录散落 |
| 工作量不可见 | 主席无法快速了解"谁在做什么、谁超负荷、谁空闲" | 缺乏跨成员聚合视图 |
| 跨部门信息割裂 | 公告发布后，关联的执行任务散落在各部门，缺少关联链路 | 公告与任务各自独立 |

### 1.2 增强目标

在不改变现有 8 个模块结构的前提下：

1. **任务模板 + 交接备注** → 知识沉淀到任务体系内，换届后人走流程留
2. **论坛知识库结构化** → 会议纪要、活动复盘、外联通讯录三套固定模板，不再依赖个人自觉
3. **主席工作看板** → 一页查看所有成员任务状态分布，一眼识别异常
4. **公告关联任务** → 校运会级别的跨部门通知直接绑定执行任务，形成"决策→执行"链路
5. **工作量可视化** → 个人中心新增热力图和排行，自我对比和横向对比

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| 模块不改结构 | 所有增强在已有文件内新增组件/函数，不动目录布局 |
| 不改路由 | 功能增强在已有页面内通过 Tab/Modal/侧栏展开 |
| 兼容已有数据 | 新增列为可选字段，有默认值，不影响已有数据查询 |
| 沿用诊断体系 | 所有新增代码使用 `logger.for(标签)`，搜标签定位文件 |
| 不改 UI 主题 | 延续 Ant Design 深蓝办公风 |


## 二、模块一：任务模板系统 + 交接备注

### 2.1 功能要点

**模板创建：**
- 部门负责人 + 主席团可创建本部门任务模板
- 每个模板包含：标题、描述、步骤清单（有序列表，每步有标题+描述）
- 模板列表页：卡片网格展示，支持编辑和删除

**从模板创建任务：**
- TaskForm 顶部新增「从模板创建」下拉选择
- 选择模板后，自动填入标题、内容（含步骤清单的 Markdown）
- 任务创建后，步骤以 Checklist 形式展示在 TaskDetail 中

**交接备注（完成时必填）：**
- 志愿者提交任务时，备注框上方新增提示"💡 填写交接备注，方便后继者"
- 部门负责人审核通过时，可追加审核备注
- 历史任务详情页展示完整备注链（提交备注 + 审核备注）

**跨部门协作标记：**
- 创建任务时，dept_head+ 可选择「关联部门」（多选），标记此任务需要哪些部门配合
- 任务卡片上显示关联部门 Tag

### 2.2 数据库变更

```sql
-- 新增：任务模板表
CREATE TABLE IF NOT EXISTS task_templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  description TEXT,
  department TEXT NOT NULL,
  steps JSONB NOT NULL DEFAULT '[]',
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- tasks 表新增列
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS template_id UUID REFERENCES task_templates(id);
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS handover_note TEXT;
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS collaborating_departments TEXT[] DEFAULT '{}';
```

`steps` JSONB 结构：
```json
[
  { "order": 1, "title": "确定节目单", "description": "收集各班级节目申报表，汇总筛选" },
  { "order": 2, "title": "审核节目", "description": "组织一审、二审，确定最终节目单" }
]
```

### 2.3 补充说明

**模板管理页位置：** TaskListPage 的页头「发布任务」按钮旁新增「📋 模板管理」按钮，点击打开 Modal 展示模板列表（卡片网格），在 Modal 内完成创建/编辑/删除，不新增路由。

**交接备注 vs 提交备注：** `task_submissions.note` 是每次提交的工作描述（可能多次提交），`tasks.handover_note` 是任务最终完成时写的一份面向后继者的交接说明（仅一份）。提交备注侧重"我做了什么"，交接备注侧重"后面的人需要知道什么"。

## 三、模块二：论坛知识库结构化

### 3.1 功能要点

在现有论坛模块内，`FORUM_CATEGORIES` 新增 `knowledge` 分类。选中该分类后，发帖时不再是自由格式，而是三选一模板：

| 模板 | 字段 | 用途 |
|------|------|------|
| 📋 会议纪要 | 会议主题、时间、地点、参会人、议题列表、决议、待办事项 | 标准化会议记录 |
| 📊 活动复盘 | 活动名称、时间、参与人数、亮点、不足、改进建议、预算决算 | 活动经验沉淀 |
| 📇 外联通讯录 | 单位名称、联系人、职务、电话/微信、合作历史、备注 | 对外关系传承 |

**权限：**
- 知识库分类下，volunteer 可查看，dept_head+ 可发布/编辑

### 3.2 数据库变更

```sql
ALTER TABLE forum_posts ADD COLUMN IF NOT EXISTS template_type TEXT;
ALTER TABLE forum_posts ADD COLUMN IF NOT EXISTS template_data JSONB;
```

`template_data` 按 template_type 存储结构化字段，例如会议纪要：
```json
{
  "meeting_time": "2026-07-01 14:00",
  "location": "学生会办公室",
  "attendees": ["张三", "李四"],
  "topics": ["迎新晚会筹备", "招新计划"],
  "resolutions": ["晚会定于9月15日", "宣传部负责海报"],
  "todos": ["张三：联系场地", "李四：设计海报初稿"]
}
```

## 四、模块三：个人中心工作量可视化

### 4.1 功能要点

在 ProfilePage 的统计面板下方，新增两个视觉组件：

**工作量热力图：**
- 类似 GitHub 贡献图，横轴为月份内的周，纵轴为周几
- 颜色深浅表示当日任务完成数，空白色表示无活动
- 默认显示当月，可左右切换月份

**任务排行榜（主席/负责人视角）：**
- 仅 dept_head+ 可见
- 展示本部门成员按本月完成任务数的排名
- 前三名高亮（金银铜）

### 4.2 数据库变更

无。从已有 `tasks` 和 `task_submissions` 表聚合查询。

## 五、模块四：成员工作看板（主席视角）

### 5.1 功能要点

权限管理页（`/admin`）新增第二个 Tab：「📊 工作看板」。

**看板内容：**

以卡片网格展示所有成员，每张卡片包含：
- 头像 / 姓名 / 部门 / 角色
- 任务状态分布柱状小图（3色：进行中 / 待审核 / 逾期）
- 当前进行中的任务数
- 逾期任务数（红色高亮）
- 点击卡片跳转到该成员的任务列表（带筛选）

**排序可切换：**
- 按逾期数降序（默认，优先关注问题成员）
- 按完成数降序
- 按部门分组

### 5.2 补充说明

**"点击卡片跳转"具体行为：** 点击成员卡片 → 导航到 `/tasks` 并传入该成员的 `assigned_to` 筛选参数，TaskListPage 接收到参数后自动筛选该成员的任务。实现方式：通过 URL query string 传递 `?member=xxx`，TaskListPage 用 `useSearchParams` 读取。

### 5.3 数据库变更

无。聚合查询 `tasks` 表。

## 九、实施顺序建议

```
阶段1：任务模板 + 交接备注（核心，改动最大）
阶段2：知识库结构化（独立，无依赖）
阶段3：工作量可视化（依赖现有任务数据）
阶段4：成员工作看板（依赖阶段3的聚合查询）
阶段5：公告关联任务（最小改动收尾）
```

每阶段完成后 `npm run build` 验证。
